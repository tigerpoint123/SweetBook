# SweetBook — 코스튬 플레이어 포토북 제작 서비스 기획서

## 1. 서비스 소개

- 코스프레 모델의 사진을 여러 팬, 사진 작가를 통해 받고 그 모델만의 포토북을 제작하여 서브컬쳐 소비자에게 판매할 수 있도록 서비스를 제공하는 프로젝트
- 타겟 수요층 : 
   1. 서브컬쳐에 관심이 있고, 코스튬 촬영·행사 참여 경험이 있는 이용자
   2. 팬 활동 사진을 정리하고 싶은 크리에이터, 소규모 서클·그룹


## 2. 실행 방법

> 


- **1차**: 코스프레·서브컬처(애니메이션·게임·만화 등)에 관심이 있고, 
- **2차**: 동인·팬 활동 사진을 정리하고 싶은 크리에이터, 
- **특성**: 시각적 표현·개성 있는 레이아웃에 민감하며, 커뮤니티 공유·보관 니즈가 큼

## 3. 주요 기능 방향 (초안)

- 등록된 코스어에게 다수의 팬, 사진 작가가 사진을 전송
- 코스어는 받은 사진을 선별
- 선별한 사진들을 코스어만의 포토북으로 제작

## 4. 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| 백엔드 (핵심 API) | **Java, Spring** | 도메인·인증·결제·주문 등 핵심 비즈니스 로직 |
| 웹 클라이언트 | **Next.js** | 사용자 웹 화면(업로드, 큐레이션, 편집, 주문/보내기) 제공 |
| 데이터 저장소 | **관계형 DB (예: MySQL/PostgreSQL)** | 회원·프로젝트·주문 등 서비스 데이터 저장 |
| 파일 스토리지 | **오브젝트 스토리지 (예: S3 호환)** | 원본 이미지·썸네일·보내기 결과물 저장 |

> 본 프로젝트는 **모놀리식 아키텍처**를 기본으로 하며, Spring 백엔드와 Next.js 웹 클라이언트를 하나의 서비스 경계로 운영한다.

## 5. 도메인 (초안)

- `member`: 회원. 계정/인증과 프로필 등 서비스 이용자의 기본 정보를 담당한다.
- `curation`: 코스튬 플레이어가 사진을 고르는 도메인. 포토북(프로젝트) 단위로 후보 이미지 중 선택 이미지(선정/제외) 및 편집을 위한 구성을 만든다.
- `campaign`: 포토북 제작에 필요한 프로세스 도메인. 촬영·행사/주문 맥락을 기준으로 업로드→큐레이션→편집→보내기(인쇄 연동이 있는 경우 주문)까지의 제작 흐름을 묶고 진행 상태를 관리한다.

---

## 6. 구현 기능 목록 (초안)

1. 기본 회원가입 및 로그인
2. 회원 중 코스어로 등록하여 포토북 기능 활성화
3. 다른 회원에게 사진 제공 받기
4. 코스어가 받은 사진 선별하기
5. 선별된 사진으로 포토북 제작
