# SweetBook — 코스튬 플레이어 포토북 제작 서비스 기획서

## 1. 프로젝트 개요

코스튬 플레이어(코스프레이어)가 촬영한 사진을 모아 **디지털·인쇄용 포토북**으로 구성·주문·공유할 수 있는 웹 서비스를 목표로 한다. 단순 갤러리가 아니라, 레이아웃·템플릿·앨범 단위 편집에 초점을 둔 **제작 도구** 성격을 갖는다.

## 2. 주제 및 가치 제안

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 코스튬 플레이어를 위한 포토북 제작 |
| **문제 인식** | 행사·스튜디오 촬영 사진이 분산되고, 인쇄물·기념 앨범으로 묶기 어려움 |
| **제안 가치** | 캐릭터·행사별 앨범 구성, 템플릿 기반 편집, (선택) 인쇄 연동으로 **서브컬처 활동의 기록·아카이빙**을 돕는다 |

## 3. 타겟 수요층

- **1차**: 코스프레·서브컬처(애니메이션·게임·만화 등)에 관심이 있고, 코스튬 촬영·행사 참여 경험이 있는 이용자
- **2차**: 동인·팬 활동 사진을 정리하고 싶은 크리에이터, 소규모 서클·그룹
- **특성**: 시각적 표현·개성 있는 레이아웃에 민감하며, 커뮤니티 공유·보관 니즈가 큼

## 4. 주요 기능 방향 (초안)

- **계정·앨범**: 사용자별 프로젝트(포토북) 단위 관리
- **이미지 업로드·정리**: 태그, 행사·캐릭터 메타데이터(선택)
- **편집**: 페이지 템플릿, 드래그 배치, 미리보기 (구체 스펙은 추후 정의)
- **보내기**: PDF·이미지 시퀀스, (선택) 인쇄 파트너 API 연동
- **공개 범위**: 비공개 / 링크 공유 등 (정책은 추후 확정)

## 5. 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| 백엔드 (핵심 API) | **Java, Spring** | 도메인·인증·결제·주문 등 핵심 비즈니스 로직 |
| 백엔드 (보조·BFF 등) | **Node.js** | 실시간·파일 처리 파이프라인, 프록시·마이크로서비스 등 역할 분리 시 활용 |
| 웹 클라이언트 | **Next.js** | SEO·라우팅·SSR/SSG가 필요한 마케팅·앱 셸 |
| **프론트엔드 (편집 UI 등)** | **임시** | 포토북 에디터·복잡 UI는 기술 선정 전 — 추후 React/Next 내 모듈 또는 별도 SPA 여부 결정 |

> 스택은 서비스 경계(모놀리식 vs 분리)에 따라 조정 가능하며, 본 문서는 **초기 방향**만 명시한다.

## 6. 리스크 및 고려사항

- **저작권·초상권**: 업로드 이미지의 권리 관계, 신고·삭제 정책
- **스토리지·대역폭**: 고해상도 원본 보관 비용
- **인쇄 연동**: 외부 업체 API·품질 기준 정의 필요

## 7. 다음 단계 (제안)

## 8. 구현 기능 목록 (초안)

아래는 본 주제(코스튬 플레이어 포토북) 프로젝트에서 **구현 대상으로 검토할 기능**을 영역별로 나열한 초안이다. 우선순위·MVP 범위는 별도로 확정한다.

### 8.1 계정·인증·프로필

- 회원 가입·로그인·로그아웃 (이메일·소셜 등 방식은 추후 결정)
- 비밀번호 재설정·이메일 인증(선택)
- 프로필(닉네임, 아바타, 소개) 조회·수정
- (선택) 크리에이터·코스튬 관련 공개 필드(작품 링크 등)

### 8.2 포토북(프로젝트)·앨범 관리

- 포토북 생성·목록·이름·설명·삭제(또는 휴지통)
- 포토북별 썸네일·상태(작성 중, 완료, 주문됨 등)
- (선택) 캐릭터명·행사명·태그로 포토북 분류·검색

### 8.3 미디어(이미지) 처리

- 이미지 업로드(단일·다중), 용량·포맷 제한·바이러스 스캔(정책에 따라)
- 원본 보관 vs 표시용 썸네일·리사이즈 파이프라인
- 업로드 진행률·실패 재시도·중복 처리(선택)
- 에디터용 에셋 라이브러리(포토북별 미디어 풀) 조회

### 8.4 포토북 편집(에디터)

- 페이지 추가·삭제·순서 변경
- 템플릿 선택·페이지별 레이아웃 슬롯에 이미지 배치
- 이미지 크롭·회전·확대 축소(뷰포트 내)
- (선택) 짧은 캡션·텍스트 박스·폰트·색상
- 실시간 미리보기·줌 인/아웃
- 편집 상태 자동 저장·수동 저장, 충돌 방지(선택)

### 8.5 보내기·인쇄

- 디지털보내기: PDF 또는 페이지별 이미지 ZIP
- 보내기 작업 큐·완료 알림(이메일 또는 인앱)
- (선택) 인쇄 규격(용지 크기, 여백, 해상도) 프리셋
- (선택) 인쇄 업체 연동: 주문 생성·결제·배송 상태 조회

### 8.7 결제·주문(인쇄 연동 시)

- 장바구니 또는 단건 주문 플로우
- 결제 연동(PG사 선택 후 구현)
- 주문 내역·취소·환불 정책에 따른 상태 관리

### 8.8 운영·신뢰·보안

- 이용약관·개인정보처리방침 동의
- 콘텐츠 신고·차단, 운영자 검토 큐(선택)
- 감사 로그·관리자 대시보드(최소: 사용자·주문 조회)
- API 속도 제한, 업로드 남용 방지

### 8.9 공통·인프라

- API 버전·에러 코드·국제화(i18n) 확장 여지
- 파일 스토리지(S3 등)·CDN 연동
- 백업·복구 전략(메타데이터·파일)
